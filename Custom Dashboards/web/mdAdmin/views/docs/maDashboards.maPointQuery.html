maPointQuery
service in module maDashboards

Description

<ma-point-query query="{$and: true, deviceName:dvName, name:ptName}" limit="5" points="points"></ma-point-query>

  • Outputs an array of data points in your Mango system.
  • The query attribute filters the array of points using an object where the keys are the property names to filter on, or supply a RQL query string inside single quotes.
  • All data points added to your Mango system will display by default, unless you set the limit property.
  • View Demo

Usage

<ma-point-query query="{name:query.filter, deviceName:query.filter}" limit="query.limit"
start="(query.page-1)*query.limit" sort="query.order" points="points" promise="promise"></ma-point-query>

Parameters

ParamTypeDetails
pointsarray

Variable to hold the array of point objects returned by the query.

queryobject

Filters the results by a property of the data points (eg: {name: 'meta'} returns data points containing the string 'meta' in the name property)

sort
(optional)
array

Sorts the resulting list by a properties of the data points. Passed as array of strings. (eg: ['-xid'] sorts descending by xid of the data points. Defaults to ['name'])

start
(optional)
number

Sets the starting index for the resulting list. Must be used in conjunction with a limit value. (Defaults to 0)

limit
(optional)
number

Limits the results in the list to a specified number of data points. Limit takes place after query and sorting (no limit by defualt)

promise
(optional)
object

Promise oibject used to tell if query data is being fetched.
