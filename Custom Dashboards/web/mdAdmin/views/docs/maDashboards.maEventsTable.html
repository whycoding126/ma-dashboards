maEventsTable
service in module maDashboards

Description

<ma-events-table></ma-events-table>

  • Displays a list of Events in a table format.
  • Allows for filtering of events by several attributes as explained below.
  • Can be set to query for events within a specific date range.
  • The table includes the ability to filter and sort by alarm level and timestamp.
  • Events can be acknowledged one at a time or a button is shown to acknowledge all events matching the query.
  • Note in usage examples below raw string literals are wrapped in single quotes where as variable names / numbers / booleans are not.
  • View Demo

Usage

<!-- Example Using filters on Table Attributes -->
<ma-events-table event-type="'SYSTEM'" alarm-level="'URGENT'" acknowledged="'*'"
active-status="'active' date-filter="true" from="fromTime" to="toTime" limit="50" 
sort="'-alarmLevel'"></ma-events-table>

<!-- Example For Restricting Events to those Related to a Data Point -->
<ma-events-table single-point="true" point-id="myPoint.id" limit="5" from="fromTime" to="toTime"></ma-events-table>

Parameters

ParamTypeDetails
limitnumber

Set the initial limit of the pagination.

point-id
(optional)
string

Filter on the Id property of a point, use with "single-point=true".

single-point
(optional)
boolean

Set to "true" and use with point-id attribute to return events related to just a single Data Point.

event-id
(optional)
number

Filter on a specific Event Id, should return a single event.

alarm-level
(optional)
string

Filter on Alarm Level. Possible values are: "'NONE'", "'INFORMATION'", "'URGENT'", "'CRITICAL'", "'LIFE_SAFETY'" or "'*'" for any.

event-type
(optional)
string

Filter on Event Type. Possible values are: "'DATA_POINT'", "'DATA_SOURCE'", "'SYSTEM'" or "'*'" for any.

acknowledged
(optional)
string

Filter on whether the event has been acknowledged. Possible values are: "'true'", "'false'" or "'*'" for either.

active-status
(optional)
string

Filter on Active Status. Possible values are: "'active'", "'noRtn'", "'normal'" or "'*'" for any.

sort
(optional)
string

Set the initial sorting column of the table. Possible values are: "'alarmLevel'", "'activeTimestamp'", "'message'" or "'acknowledged'". Precede value with a negative (eg. "'-activeTimestamp'") to reverse sorting.

from
(optional)
string

From time used for filtering by date range. Pass the value from a <ma-date-picker>.

to
(optional)
string

To time used for filtering by date range.

date-filter
(optional)
boolean

Turn on date filtering of events. Set value to "'true'" and use with from/to attribute to use. Defaults to off.

timezone
(optional)
string

Display the timestamps in this timezone
